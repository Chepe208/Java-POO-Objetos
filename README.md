Este proyecto contiene tres clases que representan objetos comunes del día a día: un carro, un libro y una nevera. El objetivo es aplicar conceptos básicos de Programación Orientada a Objetos (POO) como atributos, constructores, encapsulamiento y validaciones.

# Contenido del Proyecto

El proyecto incluye las siguientes clases:

-Carro

-Libro

-Nevera

-Main (clase principal donde se realizan las pruebas)

Cada clase tiene al menos seis atributos, métodos para acceder y modificar esos atributos (getters y setters), dos constructores (uno vacío y uno con parámetros) y un método toString para mostrar la información del objeto.

Qué hace cada clase:

**Carro**

Representa un vehículo con datos como marca, modelo, año, placa, kilometraje, tipo de combustible, precio, color y fecha de matriculación.
Incluye validaciones como el año mínimo permitido o el formato de la placa.

**Libro**

Representa un libro con información como título, autor, ISBN, número de páginas, precio, disponibilidad, fecha de publicación y género.
Incluye validaciones como el número máximo de páginas o el precio.

**Nevera**

Representa un electrodoméstico con datos como marca, modelo, capacidad, clasificación energética, precio, color, si tiene congelador y tiempo de garantía.
Incluye validaciones como la capacidad máxima o las clasificaciones permitidas.

**Clase Main**

En la clase principal se realizan las siguientes acciones:

-Se muestran objetos creados con el constructor vacío.

-Se muestran objetos creados con el constructor con parámetros.

-Se prueban las validaciones mediante valores incorrectos.

-Se demuestra el encapsulamiento al usar getters y setters para leer y modificar información.

-Se muestra un resumen final del total de objetos creados.

# Resultado

El programa imprime en consola información detallada de cada objeto, permite comprobar que los datos se asignan correctamente y muestra cómo funcionan las validaciones y el encapsulamiento.
